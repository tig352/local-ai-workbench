## V2EX

```text
分享一個剛做的本地 AI 模型工作台 MVP：Local AI Workbench。

主要解決我自己測 prompt 時的一個問題：同一段 prompt 換不同模型後，結果常常散落在聊天窗口裡，不方便比較、保存和導出。

現在這版可以：
- 保存 prompt
- 選擇 Demo / Ollama / OpenAI compatible provider
- 執行模型測試並保存記錄
- 導出 Markdown / JSON 報告
- API Key 不寫入 localStorage

還是早期版本，想先把“可復盤的本地模型測試”這個流程做紮實。
```

## 知乎回答風格

```text
如果只是偶爾聊天，普通 AI 客戶端就夠了。但如果你經常比較 prompt 或本地模型，就會遇到一個問題：測試記錄很難復盤。

我做 Local AI Workbench 的初衷就是把這件事工具化：同一個 prompt 可以對不同模型執行，結果會被保存，最後能導出 Markdown / JSON。這樣你不用靠截圖和記憶比較模型效果。

目前是早期 MVP，支持 Demo provider、Ollama 和 OpenAI compatible API。
```
